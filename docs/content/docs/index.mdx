## Overview

Reacticx is a headless, animation-focused component library for React Native.
It does **not** rely on external styling systems and is designed to integrate cleanly into existing projects.

## Setup

<Tabs items={["New Project", "Existing Project"]}>

  <Tab value="New Project">

    The fastest way to get started ‚Äî `reacticx create` scaffolds a full Expo app and wires up Reacticx in one command.

    ```bash
    npx reacticx create
    ```

    ### Interactive prompts

    The CLI will guide you through a short setup:

    ```
    ‚ú®  Create a new Expo app with Reacticx

    ‚úî App name                              ‚Ä∫ my-expo-app
    ‚úî Bundle / package ID (leave blank to skip) ‚Ä∫ com.company.myapp
    ‚úî Template                              ‚Ä∫ Blank ‚Äî TypeScript
    ‚úî Package manager                       ‚Ä∫ bun  ‚Üê detected
    ‚úî Run bunx reacticx init inside the new project? ‚Ä∫ Yes
    ```

    | Prompt | Description |
    | --- | --- |
    | **App name** | Directory name for your project. Spaces are converted to dashes automatically. |
    | **Bundle / package ID** | iOS `bundleIdentifier` and Android `package` ‚Äî written directly into `app.json`. Leave blank and it will be derived from the app name. |
    | **Template** | Starting point for your app. |
    | **Package manager** | Auto-detected from your system. You can override it. |
    | **Run reacticx init** | Initializes the Reacticx config inside the new project right after creation. |

    ### Templates

    | Template | Description |
    | --- | --- |
    | **Blank ‚Äî TypeScript** *(default)* | Minimal Expo app with TypeScript |
    | **Blank** | Minimal Expo app with JavaScript |
    | **Tabs ‚Äî Expo Router** | File-based routing with tab navigation |

    ### What happens under the hood

    ```
    ‚îå Creating Expo app
    ‚îÇ  bunx create-expo-app my-expo-app --template blank-typescript

    ‚îÇ  Updated app.json ‚Üí bundle ID: com.company.myapp

    ‚îú Initializing Reacticx
    ‚îÇ  bunx reacticx init

    ‚îî All done! üéâ

      Next steps:

        cd my-expo-app
        bunx reacticx add button
    ```

    The runner command (`bunx`, `pnpm dlx`, `yarn dlx`, `npx`) matches whichever package manager you selected.

  </Tab>

  <Tab value="Existing Project">

    Already have a React Native / Expo project? Install the required dependencies and run `init`.

    ### 1. Install dependencies

    Some components depend on shared native utilities that must be installed once.

    ```package-install
    react-native-reanimated react-native-gesture-handler react-native-svg expo-haptics expo-blur expo-symbols @expo/vector-icons @shopify/react-native-skia
    ```

    Additional dependencies are listed on individual component pages when required.

    ---

    ### 2. Initialize Reacticx

    <Card title="The config file is optional">
      `component.config.json` is only needed when using the CLI to add components.
      If you prefer copy-pasting, you can skip this step entirely.
    </Card>

    ```bash
    npx reacticx init
    ```

    You'll be prompted for an output directory (default: `src/shared/ui`).
    This creates a `component.config.json` in your project root:

    ```json
    {
      "outDir": "src/shared/ui"
    }
    ```

  </Tab>

</Tabs>

## Browse Components

List all available components:

```bash
npx reacticx list
```

Filter by category:

```bash
npx reacticx list -c molecules
```

Example output:


## Add Components

Add a component to your project:

```bash
npx reacticx add button
```

Output:

```
‚úî Added button!

Files created:
  src/shared/ui/atoms/button/index.tsx
  src/shared/ui/atoms/button/types.ts
```

### Options

| Flag           | Description                                                          |
| -------------- | -------------------------------------------------------------------- |
| `--dir <path>` | Write files to a custom directory instead of the configured `outDir` |
| `--overwrite`  | Overwrite existing files without prompting                           |

```bash
npx reacticx add button --dir src/components
npx reacticx add button --overwrite
```

## CLI Reference

| Command                                 | Description                                     |
| --------------------------------------- | ----------------------------------------------- |
| `reacticx create`                       | Scaffold a new Expo app and initialize Reacticx |
| `reacticx init`                         | Set up Reacticx in an existing project          |
| `reacticx list`                         | List all available components                   |
| `reacticx list -c <category>`           | Filter components by category                   |
| `reacticx add <component>`              | Add a component to your project                 |
| `reacticx add <component> --dir <path>` | Write to a custom directory                     |
| `reacticx add <component> --overwrite`  | Overwrite existing files                        |
